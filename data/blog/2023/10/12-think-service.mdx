## 무엇을 생각해볼것인가?
- 서비스에 대한 생각 (내가 겪은 또는 관심있는)
- 기술 스택을 도입하는데 필요한 고민
  - 기술이 등장한 이유
  - 기술을 도입하려는 이유
  - 발생할 수 있는 트레이드오프
  - 절대적인 정답은 없다.
- 좀더 소프트하게 (변수 함수 네이밍, 함수의 작성단위, 타입스크립트 작성법) - 클린코드
- 왜 라는 질문을 더 해보기
  - 왜 디자인시스템인가
  - 왜 타입스크립트인가
  - 왜 리액트인가
  - 왜 Next.js 인가 12, 13? 앱라우트? 페이지라우트?
  - 왜 코드리뷰?


  원칙과 가이드라인 없이 만든 프로덕트는
  ui 일관성을 떨어뜨리고 사용자에게 혼란을 주거나 잘못된 방향으로 학습시킨다.
  버튼을 만드는방법(버튼처럼보이게 만드는 방법)
  ```html
  <button class="btn-red btn-size-lg">click</button>
  <a class="btn-red btn-size-lg">click</a>
  <div class="btn-red btn-size-lg">click</div>
  ```

  조직이 변하면 프로덕트도 영향을 받는다
  조직의 변화와 무관한 프로덕트를 만들수는 없을까

  시스템 (운영체제)
  - 복잡한체계 (하드웨어와 소프트웨어 환경)
  - 구조화 행동 (CPU, 메모리, 디스크 등 리소스 사용)
  - 통제하는 규칙 (스케쥴러, 가상메모리, 파일시스템)

  네트워크 시스템
  - 복잡한체계 통신설비로 서로 연결된 환경
  - 데이터 전송 구조화행동
  - 통제하는 규칙 TCP/IP HTTP SSL MIME

  당연한 UX
  일관성있는 경험을 줄수 있는 UI
